# yidiancst.ai 内容改版设计 (2026-05-19)

> 本文件为设计 spec。批准后转 writing-plans 出实施计划，再进入实现。

## 1. 目标与背景

- **站点**：yidiancst.ai（Vercel 项目 `yidian-website`，源 `~/yidian-website`，改 main 即自动部署）。
- **现状**：最后内容更新 2026-04-07，距今约 6 周；早于品牌 v2（4-27），且未体现知微近 6 周演进。
- **本次目标**：按知微当前真实形态，做一次**内容 + 结构 + 品牌 v2 视觉**三层更新。
- **首要受众**：危废服务商 / 排污（产废）企业（知微 Pro 获客向）。
- **不做**：信息架构重设计、视觉重做、后端/表单系统、实时试用系统接入、ECS `yidiancst.com.cn`（ICP 线）改动、联系方式/地址变更。

## 2. 知微对外定位（ground truth · 已据实核实）

知微 = 危废合规审查辅助产品，对外呈现**两大并列主模块 + 支撑能力**：

- **主模块① 危废规范化管理 · 合规自查与评估准备辅助**（政府部门最先投入使用的主力线）
- **主模块② 危废鉴别 · 复核辅助**
- **支撑能力**：跨资料一致性比对 / 现场核查（OCR+视觉）/ 全程本地可控

### 公开站零出现（硬约束）

- **AI 打废 / dafei / "打废" 字样**——整体隐藏，全站零出现。
- 具体政府客户名（东莞局 / 四会分局 / 肇庆 / 某环保局 / 华南所）。
- 具体项目指标（97.6% / 90 家 / 院士 GT / 54 家 等）。
- 内部模型架构（三级路由 / 9b·27b / 算力中心 / provider 名）。
- 备案号、算法/ICP 备案"已完成"类表述。

### 政府客户：可提，必须脱敏（用户 2026-05-19 授权）

- ✅ 可写："已为**地市级生态环境主管部门**提供危险废物规范化管理评估技术支撑"。
- ✅ 可写："服务于政府监管侧与产废企业双场景"。
- ❌ 不得出现：任何城市名、局/分局名、合作机构名、项目名、项目数值指标。

## 4. 叙事与视觉风格（Apple / Anthropic 调性）

唐总指定：贴近 Claude（Anthropic）与 Apple 的叙事——**简洁、凝练、克制而高大上**。落地准则：

**文案**
- 标题短、陈述句、讲结果不堆功能；副题一句有信息量的话；正文砍形容词。
- **零 AI 黑话**：禁"赋能 / 抓手 / 闭环 / 生态 / 一站式 / 全方位"等（同 `feedback_expert_review_package_standard`、`feedback_ai_narrative_principle`）。
- 讲价值用"传统人工做不到 X，知微让 Y 成为可能"的**冷静对照**，不喊口号。
- Anthropic 式信任叙事：把"辅助而不替代专家""数据不出本地"当作**正面卖点**讲，而非免责小字。

**视觉（品牌 v2 对齐）**
- 主渐变 `#05B27D→#31D57A` **仅用于强调**（主 CTA、关键强调字、logo）；其余近单色：`#1A1A1A` 主文 / `#6B7280` 次文 / `#F7F8FA` 背景 / `#E5E7EB` 描边。
- **去掉**现状满屏 emerald 渐变、彩色徽章墙、数字墙（stat wall）。
- 放大 Hero / Section 标题字号 + 大留白；正文收紧、行距宽松。
- 组件克制：双主模块用 alternating 大块"feature moment"，支撑能力用等权小三栏。
- 页脚副标 `ZHIWEI` 取代 `YIDIAN CST` + slogan「知微见著 · 洞察每一点价值」+ 性质声明。
- Logo：品牌 v2 图形资产在 `zhiwei-desktop` 仓，本官网仓**无 v2 logo 文件**——本次先对齐**配色 + 文字标 + slogan**；v2 logo 图形资产迁入列为**明确 follow-up**（不阻塞本次上线，spec 末「遗留项」记录）。

## 5. 信息架构与内容（区块逐节）

> 下列文案为**推荐定稿**，spec 评审时可逐句替换。

**(1) 顶部导航**：首页 / 产品 / 双模块锚点 / 关于 / 联系（精简，去冗余项）。

**(2) Hero**
- 品牌行：知微 · Zhīwēi（保留 易经「知微知彰」DNA，一行小字）
- H1（推荐）：**"看清每一处合规偏差。"**（备选：「危废合规，见微知著。」/「把合规，做在检查之前。」）
- 副题：「知微是危险废物规范化管理与鉴别复核的智能审查助手——在海量台账、申报、联单与鉴别报告中，主动发现关键偏差与证据缺口。」
- CTA：主「申请试用资格」+ 次「预约演示」（**不写"立即试用"**，公网自助试用 ICP-gated）

**(3) 主模块① 危废规范化管理 · 合规自查与评估准备**
- H2（推荐）：「规范化管理，全项自查不漏项。」
- 正文：「对照国家危险废物规范化环境管理评估要求，知微逐项核对台账、申报、转移联单、贮存与标识、管理计划、应急与培训，定位每一处差距与缺证，产出可整改的自查底稿。」
- LLM 对照句：「人工逐档比对，难免漏项、口径不一；知微跨多份非结构化材料语义对齐，让全项自查与证据归集第一次成为常态。」

**(4) 主模块② 危废鉴别 · 复核辅助**
- H2（推荐）：「鉴别复核，专业初审更稳。」
- 正文：「围绕属性判定与鉴别报告，知微辅助核查采样代表性、检测项目与标准适用性，沿问题树组织证据链。」
- 红线声明（正面表述）：「知微辅助初审，不替代专家最终判定，不输出执法或定性结论。」

**(5) 支撑能力**（等权小三栏）
- 跨资料一致性比对：台账 × 排污许可 × 环评 × 联单，自动定位互相矛盾处。
- 现场核查辅助：OCR + 视觉双通道，照片标签/容器状态与台账联动。
- 全程本地可控：核心模型与敏感资料留在本地环境，数据不出域。

**(6) 为什么是知微 / 信任**（Anthropic 式三柱，克制）
- 一线沉淀：源自环保咨询一线的方法，沉淀为可复用的审查逻辑。
- 真实验证（脱敏）：已为地市级生态环境主管部门提供危废规范化管理评估技术支撑，并服务于产废企业与危废服务商。
- 可控与克制：模型与敏感资料全程本地；知微只做辅助，最终判断留给专家。

**(7) 能力进展 / Roadmap**（据实，不通胀）：台账智能审查与跨资料比对已上线 → 现场核查与鉴别复核完善中 → 全过程合规审查持续演进。

**(8) 关于亿点**：广州市亿点环保有限公司；环保咨询服务底盘（原 12 项收敛为简洁分组列表，非 12 张卡墙）；克制的资质背书（软著 5 / 实新 8 / 发明 2 优先审查中；产学研：广东工业大学）。

**(9) CTA / 联系**：收尾一句（推荐：「把合规细节交给知微，把判断留给你。」）+「申请试用资格」/「预约演示」+ 现有联系方式（电话/邮箱不变）。CTA 走 `mailto`/电话锚点或既有联系区，**不新建后端 lead 系统**。

**(10) 页脚**：ZHIWEI + slogan + 性质声明（「本产品为 AI 辅助参考工具，不构成行政执法决定，最终以专家核查为准。」）+ 公司全称 + 动态版权年。

**(11) SEO metadata / OG / JSON-LD**：同步双模块叙事 + ZHIWEI；title/description 重写为"危废规范化管理与鉴别复核智能审查"；**不写备案号**；Organization schema 公司名/联系不变。
